# SCOTUS Cross‑Ideological Alignment — Part 3 (Closely Divided Cases)

Replication code for:  
**“Who Crosses Over in Close Cases?”**

This repository reproduces the Part 3 analysis of cross‑ideological voting behavior among Supreme Court justices from the 2020 Term through the 2024 Term, restricting attention to closely divided cases.

---

# Overview

Parts 1 and 2 examined cross‑ideological alignment across all cases and across majority vs. dissent votes.  
Part 3 narrows the focus to the cases where crossover matters most — closely divided decisions:

- **Close** cases: the majority had 5 or 6 votes (5–4 and 6–3 decisions)
- **Very close** cases: the majority had exactly 5 votes (5–4 decisions)

For each ideological bloc the analysis estimates each justice’s probability of voting on the same side as the opposing bloc, separately for close and very close cases.

Models are estimated using logistic regression with case‑level clustered standard errors.

---

# Data

The analysis relies on the **justice‑centered dataset** from the Supreme Court Database (SCDB).

To replicate:

1. Download the justice‑centered dataset from:  
   http://scdb.wustl.edu/index.php
2. Place the dataset file in the root directory of this repository.
3. Ensure the file name matches:  
   `SCDB_2025_01_justiceCentered_Citation.dta`

The raw dataset is not included in this repository.

# Replication Instructions

To reproduce all results:

1. Open Stata (version 18 recommended).
2. Navigate to the repository root directory.
3. Run:

```
do code/00_masterfile.do
```

This will:

- Construct pairwise datasets restricted to closely divided cases  
- Estimate close‑case and very‑close‑case alignment models for each bloc  
- Run pairwise comparisons across justices  
- Generate dot‑and‑whisker plots  
- Export figures to `output/figures/`

---

# Methods Summary

- **Sample**: 2020–2024 Terms, closely divided cases  
- **Unit of analysis**: justice‑pair vote  
- **Alignment**: voting on the same side of the case (both majority or both dissent)  
- **Closeness**:
  - Close = majority of 5 or 6 votes (5–4 or 6–3)
  - Very close = majority of exactly 5 votes (5–4)  
- **Estimation**: logistic regression  
- **Standard errors**: clustered by case  
- **Visualization**: margins + marginsplot
